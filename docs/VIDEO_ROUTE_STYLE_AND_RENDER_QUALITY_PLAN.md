# 视频路线风格与渲染质量规划

> 视频最终渲染效果是产品核心。
> Claude 遗留任务不是旁支，而是渲染质量优化主线。
> Style Gallery 不是替代渲染优化，而是承载优化结果的入口。

---

## 1. 产品定位

**视频生成技术路线风格实验台** — 不是"一键生成视频工具"，而是探索不同视频生成技术路线的风格效果、能力边界、渲染质量优化和复用价值的平台。

核心问题是：**每条技术路线天然适合什么风格？每条路线的魅力和边界是什么？哪些渲染效果真正值得保存并长期复用？**

---

## 2. 当前阶段判断

当前处于**风格样片库**阶段（V0.3.7）：

1. 每条路线可以生成代表性风格样片
2. 每个样片保存路线、风格、参数、输出文件
3. 支持查看、删除、加入对比
4. **暂不正式升级为模板**
5. **暂不做路线混合**
6. **暂不做自动路线选择**

---

## 3. Claude 遗留任务关系

Claude 遗留任务不是旁支，而是**渲染质量优化主线**。与 Style Gallery 的关系：

| 遗留任务 | 优先级 | 与 Style Gallery 关系 |
|---|---|---|
| P0 数据可视化 | 基本完成 | 作为样片库基础能力，showDataViz 控制开关 |
| P1 BGM + 音效 | 后续继续 | 直接影响最终分享感，在样片中体现效果 |
| P2 Remotion 动效增强 | 后续继续 | 体现 Remotion 技术魅力，动效强的样片优先探索 |
| 视觉评分 / 质量历史 | 后续接入 | 每个样片可记录质量分和优化建议 |
| 自动优化回路 | 暂缓 | 依赖样片库积累足够数据后再做 |

**结论**：BGM、Remotion 动效增强不能丢，是渲染质量的重要组成部分。

---

## 4. 渲染质量优化主线

渲染质量是**产品核心能力**，是 Style Gallery 的底层支撑：

```
渲染质量优化（Claude 遗留任务主线）
    ↓ 产出高质量样片
Style Gallery（样片沉淀入口）
    ↓ 保存、查看、对比
参数 / preset 系统（后续复用层）
    ↓ 优秀样片升级
模板保存（V0.3.9）
```

当前只做渲染质量优化 → Style Gallery，不跨阶段做模板。

---

## 5. Style Gallery 入口定位

Style Gallery 是**样片沉淀入口**，不是替代渲染优化，也不是复杂模板系统。

**职责**：
- 保存每次风格探索的结果（路线 + 风格 + 参数 + 输出文件）
- 支持按路线筛选、查看、删除、加入对比
- 记录人工评价（可读性、动态感、视觉冲击、稳定性、成本）
- 展示预置风格入口（每条路线 ≥ 2 个风格）

**不做**：
- 路线混合
- 自动选择路线
- 复杂模板保存（延后到 V0.3.9）
- 复杂 preset 系统（延后）
- 数据库

---

## 6. Style Sample 数据对象

最小字段：

```json
{
  "id": "sample_xxx",
  "route_id": "template_programmatic_render",
  "route_name": "Remotion 动态模板路线",
  "style_name": "动态数据栏目",
  "description": "数字滚动 + 数据条增长，展示 Remotion 时间轴动画能力",
  "status": "candidate",
  "params": { "showDataViz": true, "metricAnimation": "countup_bar", ... },
  "output": { "type": "mp4", "path": "...", "poster": "..." },
  "evaluation": { "readability": 4, "motion": 5, "visual_impact": 4, "notes": "" },
  "created_at": "2026-06-15T00:00:00Z"
}
```

**状态流转**：`candidate` → `comparing` → `approved` / `rejected`

**存储**：JSONL 文件，不引入数据库。

## 8. 暂不做事项

1. **不做路线混合** — 每条路线独立生成，不拼接
2. **不做自动选择路线** — 用户自己判断哪条路线适合
3. **不做多路线拼接成一个视频** — 各路线独立
4. **不新增视频路线** — 当前只有 Pillow / Remotion / AI素材
5. **不做数据库** — JSONL 足够
6. **不做任务队列** — 同步返回
7. **不重构 FFmpeg / TTS 主流程** — 复用现有链路
8. **不做 route profile 大包装页** — V0.4.0 多路线对比时再考虑

## 10. 三条路线定位（备忘）

| 路线 | 定位 | 核心魅力 |
|---|---|---|
| **Pillow** | 稳定信息卡 / 批量生成 / 兜底路线 | 稳定排版、长文本可读、数据卡清晰 |
| **Remotion** | React 时间轴动画 / 动态栏目模板 | 数字滚动、进度条生长、卡片入场、页面转场 |
| **AI素材** | AI 生成背景 / 氛围视觉 / 展示型短视频 | 科技感氛围、封面感、背景+信息卡融合 |
