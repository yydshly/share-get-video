# 视频生成技术路线探索纲领

> 创建日期：2026-06-14
> 目的：系统化探索"一个想法 → 高质量分享视频"的能力。
> 与 `VIDEO_METHODS_MATRIX.md` 的区别：矩阵回答"有哪些方式、各自适合什么"；
> 本文回答"**每种方式如何推到它的画质天花板**，以及在我们的场景下到底哪条路线最优"。
>
> 探索原则：**不止跑通，要推到最佳状态**。每条路线的评估必须基于"已尽力优化"的产物，
> 而不是 demo 级首版。否则对比结论无效。

---

## 0. 锚定场景

探索必须先锚定场景，否则"哪种最好"无意义。

| 场景代号 | 描述 | 规格 | 核心诉求 |
|---------|------|------|---------|
| **S1 AI 资讯分享**（主） | AI 信息总结后做分享短视频 | 9:16，30-45s | 信息准确、文案有料、可批量、字幕可控 |
| S2 情绪/氛围短片 | 强视觉冲击的氛围内容 | 9:16/16:9，10-20s | 画面冲击力、运动自然 |
| S3 知识讲解 | 较长的讲解视频 | 16:9，2-5min | 结构清晰、口播自然、信息承载 |
| S4 图片动起来 | 单图/少量图动态化 | 任意 | 运动自然、低成本 |
| S5 产品介绍 | 品牌调性一致的介绍片 | 16:9/9:16，30-60s | 一致性、精致度、可控 |

本纲领以 **S1** 为主线，其余场景作为对比参照。

## 第一步：生成视频的方式全景

按"画面从哪来"分三大家族，外加所有路线共享的横切能力层。
括号内为本仓库对应的 `MethodCategory` / chain。

### 家族 A — 程序化确定性渲染（画面由代码精确控制）

| # | 路线 | 技术栈 | 仓库对应 | 现状 |
|---|------|--------|---------|------|
| A1 | 本地帧合成 | Pillow/OpenCV 画帧 + FFmpeg 拼接 | `local_frame_compose` | Real ✅ |
| A2 | 代码模板渲染 | Remotion(React) / Motion Canvas / Lottie | `template_programmatic_render` | Real（Win 挂） |
| A3 | HTML/CSS 渲染 | HTML+CSS 动画 → 录屏 / HeyGen HyperFrames | `hyperframes_html_render` | Manual |
| A4 | 本地素材剪辑 | 已有视频/图片片段 + MoviePy/FFmpeg 编排 | `local_media_compose` | Mock |

特征：**文字 100% 精确、可复现、可批量、成本近零**；画面表现力依赖模板设计与动画功底。

### 家族 B — AI 生成式（画面由模型生成）

| # | 路线 | 代表能力 | 仓库对应 | 现状 |
|---|------|---------|---------|------|
| B1 | AI 视频直出 | 文生/图生视频：可灵 Kling、Runway Gen-4、Google Veo、Sora、海螺 Hailuo、Pika | `ai_video_direct` | Reserved |
| B2 | AI 数字人口播 | HeyGen / Synthesia / 即梦 数字人 | （未单列，可归 ai_asset） | 未接 |
| B3 | AI 素材 + 程序化合成 | LLM 脚本 + AI 图/视频 B-roll + TTS + 程序化拼装 | `ai_asset_then_compose` | Mock |

特征：**画面表现力天花板高、运动自然**；但文字不可控、一致性差、成本高、有随机性。

### 家族 C — 混合编排

| # | 路线 | 思路 | 仓库对应 | 现状 |
|---|------|------|---------|------|
| C1 | 混合流水线 | 按段落性质自动选手段（信息段用程序化，氛围段用 AI 直出） | `hybrid_pipeline` | Mock |

特征：理论最优，工程复杂度最高，应在 A/B 各自摸清天花板**之后**再做。

### 横切能力层 L（所有家族共享，决定内容质量上限）

| 层 | 作用 | 仓库现状 | 关键问题 |
|----|------|---------|---------|
| **L1 内容/脚本层** | 想法 → 结构化 → 脚本 → 分镜 → 文案 | 模板拼接，**未接 LLM** | ⚠️ 地基缺失，详见下方 |
| L2 配音 TTS | 文本 → 语音 | MiniMax TTS，Real ✅ | 可换音色/语速探索 |
| L3 字幕 | 时间轴字幕 + 烧入 | SRT/ASS，Real ✅ | 样式可继续打磨 |
| L4 BGM/音效 | 背景音乐、转场音效 | 未接 | 影响"分享感" |

> **关键认知**：L1 是所有路线的共同上游。它现在是模板拼接（`voiceover_planner.py` 明确
> "Does NOT call LLM"），意味着无论下游用哪条路线，喂进去的"内容"质量都被这层封顶。
> **不先打通 L1，任何路线的对比都是在比"谁能把平庸文案渲染得好看一点"，结论不可信。**

## 第二步：每种方式推到"最佳状态"的技术探索

每条路线给出：**画质天花板 → 推到最佳的关键杠杆 → 在本仓库的落地点 → 已知失败点**。
探索这条路线时，必须把"关键杠杆"全部尝试过，才算"已尽力优化"，其产物才可进入第三步对比。

### L1 内容/脚本层（先做，所有路线受益）

- **天花板**：决定了文案是"第1点，第2点"式机械播报，还是有钩子、有节奏、有观点的口播稿。
- **关键杠杆**：
  1. 用 LLM（Claude）替换 `script_planner` / `voiceover_planner` 的模板逻辑，输入"AI 信息总结"，输出：①一句话钩子开场 ②每条资讯的"是什么+为什么重要"重写 ③金句收尾。
  2. 分镜结构化输出（每个镜头：时长、画面要点、字幕文案、旁白文案、B-roll 提示词），作为下游所有路线的统一输入契约。
  3. 文案与画面解耦：旁白稿（口语化）与屏幕字幕（精炼）分别生成，而非同一份。
- **落地点**：`planners/script_planner.py`、`planners/voiceover_planner.py`、`planners/storyboard_planner.py`；建议新增统一产物 `ShotPlan`（分镜契约）。
- **失败点**：LLM 幻觉改写事实 → 需加"忠实于原文事实"约束 + 关键数字保留校验。

### A1 本地帧合成（当前唯一全跑通，画面 2/5）

- **天花板**：本质是"会动的信息卡片"。上限约 3.5-4/5——做不到电影感，但能做到"干净、专业、信息密度高、节奏好"。
- **关键杠杆**：
  1. **动态化**：静态帧 → Ken Burns（缓慢缩放/平移）、文字逐行入场、数字滚动、进度条增长。这是从 2/5 到 3.5/5 的最大单点。
  2. **视觉系统**：统一栅格、字阶、配色（已有 `visual_theme.py`），加品牌元素、图标、分隔装饰。
  3. **数据可视化**：把"提升 40%"渲染成实际的条形/环形增长动画，而非纯文字。
  4. **转场**：已有 fade（`transition_composer.py`），可加位移/遮罩转场。
  5. **背景层次**：渐变 + 噪点 + 模糊光斑，避免纯色死板。
- **落地点**：`renderers/frame_templates.py`、`local_frame_renderer.py`、`transition_composer.py`（动画需改为"多帧序列"而非单帧）。
- **失败点**：逐帧生成大量帧时性能 →（已部分修过）；中文排版换行/标点。

### A2 代码模板渲染 / Remotion（最有潜力的程序化路线，当前 Win 挂）

- **天花板**：约 4-4.5/5。React 全部 CSS/SVG/Canvas 能力 + 帧级动画曲线（spring/easing），能做出接近专业 MG 动画的资讯片，且文字 100% 精确、可批量。**S1 场景的理论最优程序化路线。**
- **关键杠杆**：
  1. **先修环境**：当前 `FFmpeg quit with code 3752568763`（Windows 兼容）阻断一切。优先排查 Remotion 的 ffmpeg 路径/参数、`--gl` 渲染后端、并发设置。
  2. 用 `spring()` / `interpolate()` 做物理感入场动画。
  3. 组件化模板：Cover / KeyPointCard / DataViz / Summary，props 由 L1 的 ShotPlan 驱动。
  4. 高级排版：可变字体、emoji、SVG 图标、Lottie 嵌入。
  5. 音频同步：把 TTS 时间轴喂给 Remotion，画面节奏卡配音。
- **落地点**：`remotion/src/AiNewsVideo.tsx`、`renderers/remotion/`；`chains/remotion_tts_video.py`。
- **失败点**：①Windows FFmpeg 退出码（**头号阻断**）②渲染慢（45s 视频数分钟）③Node 环境依赖。

### A3 HTML/CSS → HyperFrames（manual，未实测上限）

- **天花板**：取决于 HeyGen HyperFrames 对 HTML/CSS 动画的还原度，未实测。理论上接近 A2。
- **关键杠杆**：
  1. **先人工实测一次**：把已生成的 `index.html` 真正放进 HeyGen 渲染，看画质上限和限制（CSS 支持范围、时长、水印）。
  2. 若还原度高 → 可作为"无需 Node/FFmpeg 环境"的轻量替代。
- **落地点**：`renderers/hyperframes/html_builder.py`；`chains/hyperframes_tts_video.py`。
- **失败点**：依赖第三方插件、需人工、CSS 子集限制、不可全自动批量。

### A4 本地素材剪辑（Mock，需先决定是否值得做）

- **天花板**：取决于素材库质量。有好素材时可达 4/5，但 S1 场景通常没有现成对口素材。
- **关键杠杆**：素材库 + 智能选片 + 节奏剪辑。对 S1 优先级低，对 S4（图片动起来）/有素材积累时再做。
- **落地点**：`adapters/local_media_compose.py`（当前返回 mock）。

### B1 AI 视频直出（Reserved，画面天花板最高但文字不可控）

- **天花板**：画面 4.5-5/5（运动、质感、氛围），但**承载精确信息 1-2/5**（字幕错漏、数字乱）。
- **关键杠杆**：
  1. **定位为 B-roll 而非主体**：用它生成背景/氛围镜头，文字信息靠程序化层叠加（这是与 A2 结合的 B3 思路）。
  2. 图生视频优于文生视频（可控性更高，用 AI 生成的封面图驱动）。
  3. 首尾帧控制运动起止。
  4. 选型对比：可灵 Kling（性价比/中文友好）、Runway Gen-4（运镜）、Veo（质感）、海螺 Hailuo。
- **落地点**：`adapters/ai_video_direct.py`（当前 Reserved）。
- **失败点**：成本高、慢、随机性、字幕完全不可控、批量一致性差。

### B3 AI 素材 + 程序化合成（Mock，工程量大但综合最优潜力）

- **天花板**：综合 4.5/5。AI 生成封面/背景/B-roll（好看）+ 程序化叠精确文字字幕（可控）+ TTS（自然），取各家之长。
- **关键杠杆**：
  1. LLM（L1）出脚本 + 图像提示词。
  2. 文生图（Flux/SDXL/即梦/DALL-E）出封面与背景。
  3. 可选 B1 让关键镜头动起来。
  4. A1/A2 叠加精确字幕与信息层。
  5. TTS + 字幕 + BGM 合成。
- **落地点**：`adapters/ai_asset_then_compose.py`（当前 mock）；强依赖 L1。
- **失败点**：多阶段串行（10-30min）、成本高、任一环节失败影响整体 → 需健壮降级。

### B2 AI 数字人口播（未接，S3 友好）

- **天花板**：S3 知识讲解 4/5（有"人"在讲，亲和力强）；S1 短资讯反而可能拖慢节奏。
- **关键杠杆**：HeyGen/Synthesia 数字人 + L1 口播稿 + 画中画信息卡。
- **失败点**：成本、数字人"恐怖谷"、品牌一致性。

### C1 混合编排（最后做）

- 在 A2 与 B1/B3 各自摸清天花板后，按 ShotPlan 的镜头类型自动路由：信息密集段→A2，氛围段→B1。
- 落地点：`adapters/hybrid_pipeline.py`。**过早做会是空中楼阁。**

## 第三步：对比与场景适配

### 3.1 统一评测维度（沿用 `VideoExperimentEvaluation`，1-5）

信息准确性 / 字幕可读性 / 画面质量 / 节奏感 / 分享意愿 / 自动化稳定性 / 成本可控性 / 产品化潜力。

> **对比有效性前提**：每条路线必须用"已尽力优化"的产物参评（第二步杠杆都试过），
> 且喂同一份 L1 输出。当前评估报告（V0.3.5）只有 1 条真跑通、画面是首版，**结论暂不可作数**。

### 3.4 探索执行顺序（建议）

```
阶段一 打地基（让对比成立）
  1. 打通 L1 LLM 内容层（想法→ShotPlan）        ← 所有路线受益
  2. 把实验+评分持久化（SQLite）                  ← 否则对比数据一重启就丢
  3. 统一"已尽力优化"产物 + 同输入评测口径

阶段二 把两条程序化路线推到最佳
  4. A1 本地帧动态化（2/5 → 3.5/5）
  5. 修 A2 Remotion Windows 环境 → 推到 4/5
  6. A1 vs A2 横向实测对比，定 S1 主推

阶段三 引入 AI 画面
  7. B1 接一个模型（可灵/Runway）做 B-roll
  8. B3 = L1 + AI图 + B1 + A2叠字幕 + TTS，做增强链路
  9. A2 vs B3 对比，定"主推 vs 增强"分工

阶段四 终局
  10. C1 混合编排（按镜头路由）
  11. 形成场景→路线决策表（替换 3.3 的"假设"为"实测结论"）
```
