# V0.6.1 Remotion 表现范式探索

## 背景

V0.6.0 已完成三路线（Pillow / Remotion / AI素材）baseline 对比面板。

当前架构判断：

```
Remotion 不应该只作为一条单一路线，
而应该作为一个可编程视频表现系统。
```

也就是：

```
Remotion = 多种视频表现范式的承载引擎
```

## 为什么 Remotion 不是单一路线

当前 Remotion 路线（`template_programmatic_render`）在 baseline 对比中定位为"动效精品模板路线"。

但 Remotion 的本质是一个**可编程视频渲染引擎**，同一个引擎可以承载多种完全不同的视觉表现范式：

- 同一个 AiNewsVideo 可以演化出 Data News、Card Stack、Timeline 等不同子范式
- 不同子范式在视觉表达、适用内容、实现难度上差异巨大
- 把它们混在一条"Remotion 路线"下会导致：定位模糊、无法对比、难以规划

因此，有必要把 Remotion 拆解为**可探索的多表现范式家族**。

## 五种表现范式

### 1. Data News（P0）

**定位**：数据新闻 / AI资讯指标变化 / 百分比高亮

**适合内容**：
- 模型性能提升
- Benchmark 通过率
- 融资/估值数据
- 用户增长指标
- 榜单变化

**视觉特征**：
- 数字滚动动画
- 数据条生长
- 百分比高亮
- 趋势箭头
- 指标卡片

**当前状态**：已有基础 — AiNewsVideo 当前形态最接近此方向

**推荐优先级**：P0

**P0 原因**：与当前 AI 新闻场景最贴近；已有基础；风险最低

---

### 2. Card Stack（P1）

**定位**：卡片堆叠 / 短资讯合集 / 短视频信息流

**适合内容**：
- AI 工具推荐
- 今日 AI 三件事
- 多产品快速介绍
- 模型能力清单
- 短资讯合集

**视觉特征**：
- 卡片滑入出场
- 当前卡片突出
- 上一张退到背景层
- 下一张卡片预览
- 短视频节奏

**当前状态**：待探索 — 需在现有 AiNewsVideo 基础上增加 variant

**推荐优先级**：P1

**P1 原因**：短视频感最强，适合 AI 新闻信息流方向

---

### 3. Timeline（P1）

**定位**：事件演进 / 时间线 / 因果顺序

**适合内容**：
- 政策演进
- 产品发布步骤
- 论文方法流程
- 技术路线变化
- 公司合作进展

**视觉特征**：
- 竖向时间线
- 节点依次出现
- 当前节点高亮
- 历史节点淡化
- 因果箭头连接

**当前状态**：待探索 — 需要 Timeline 专用组件

**推荐优先级**：P1

**P1 原因**：适合解释复杂事件，与 Data News 正交

---

### 4. Dashboard（P2）

**定位**：多指标看板 / 排行榜 / 模型对比矩阵

**适合内容**：
- 模型排行榜
- Benchmark 矩阵
- 多公司指标对比
- 产品能力评分
- 投资组合概览

**视觉特征**：
- 多数据面板
- 排行榜
- 进度条
- 矩阵布局
- 小型指标组件

**当前状态**：待探索 — 复杂度较高，需谨慎投入

**推荐优先级**：P2

**P2 原因**：信息密度高但复杂度也高，容易过度设计

---

### 5. Subtitle Story（P2）

**定位**：旁白驱动 / 大字幕 / 情绪叙事

**适合内容**：
- 观点短片
- 深夜独白
- 情绪 MV
- 一句话长成短片
- 口播内容

**视觉特征**：
- 大字幕占主导
- 弱化画面
- 背景轻动效
- 节奏跟随旁白
- 情绪色彩渐变

**当前状态**：待探索 — 更适合另一个文案/情绪视频产品线

**推荐优先级**：P2

**P2 原因**：更偏向情绪内容，与当前 AI 新闻定位差异较大

## 推荐推进顺序

### 1. 先巩固 Data News（P0）

理由：与当前 AI 新闻场景最贴近；已有基础；风险最低

### 2. 然后探索 Card Stack（P1）

理由：
- 短视频感最强
- "今日 AI 三件事"场景明确
- 可快速验证
- 与 Data News 互补

### 3. 再探索 Timeline（P1）

理由：
- 适合解释复杂事件
- 与 Data News 正交
- 拓展内容边界

### 4. Dashboard / Subtitle Story 后置（P2）

理由：
- Dashboard 复杂度高，容易过度设计
- Subtitle Story 更适合情绪视频产品线，与当前 AI 新闻定位差异较大

## 是否生成最小样片

**本轮执行方案 A（UI 规划优先）**。

本轮仅完成 Remotion 表现范式的 UI 和文档沉淀。

Card Stack 最小样片留待下一轮实现。

**下一轮 Card Stack 最小实现方案**：

在现有 `AiNewsVideo.tsx` 增加轻量参数 `remotionFamily?: "data_news" | "card_stack"`：

```ts
当 remissionFamily === "card_stack" 时：
- KeyPointCard 改为全屏布局
- 上一张卡片作为背景层缩小/淡出
- 当前卡片突出显示
- 下一张卡片滑入预览
- 不重写整个渲染系统
```

目标：用 3 条 AI 新闻生成卡片堆叠式短片样片。
